### 6 — Map スコアへのハウス考慮（2026-04-25 議論・別セッションで実装予定）

**経緯:**
2026-04-25 に Horo 占い文へのハウス考慮（Phase H1+H2）を実装した際、オーナーから「惑星配置がハウスに大きく関係している場合、Map の方位スコアにも影響が出るのでは？」と指摘あり。検証の結果、現状の Map スコアは **ハウスを weight に反映していない** ため改善余地ありと確認。

**現状の Map スコア計算（map_astro.dart `scoreAll`）:**
- 各惑星の絶対黄経 → 16方位との角度差で減衰（cosFall）
- 各惑星のアスペクト weight を掛け合わせ
- → **ハウス位置が惑星の影響力（weight）を変えていない**

#### Phase M1 — natal house weight 付与（軽量改修・1時間）

各惑星の natal house を読んで、カテゴリ別スコア計算で weight 付け：
- love → 7H/5H 在住の金星・火星・月に 1.5倍 weight
- money → 2H/8H 在住の金星・木星・土星に 1.5倍 weight
- career → 6H/10H 在住の太陽・土星・金星に 1.5倍 weight
- communication → 3H/9H 在住の水星・月・木星に 1.5倍 weight
- healing → 12H/4H 在住の月・海王星・金星に 1.5倍 weight

**実装範囲:**
- `scoreAll()` シグネチャに `Map<String, int> natalHouses` を追加
- `_fortunePairs` の構造を拡張、または新規 `_houseWeights` テーブル
- 既存の 16方位スコア体系は変えない、weight のみ追加

**効果:**
- カテゴリ別占いの精度向上
- 伝統占星術寄りの評価
- 出生時刻不明ユーザーは現状のまま

#### Phase M2 — 真のアストロカートグラフィー（大改修・1〜2日）

地理座標を渡すと **その地点でのハウス再計算** を行い、惑星のハウス位置を変える：
- 例：東京で 7H にあった金星が、ハワイでは 5H に移動する
- 行き先によって惑星の意味とテーマが変わる
- 本物の地理位置占星術になる

**実装範囲:**
- Worker `/astro/chart` に `targetLat/targetLng` パラメータ追加
- 行き先座標で houses 配列を再計算（既存 calcHouses を流用）
- 行き先での planetHouses を再算出
- Map 上の任意地点でハウス変化を可視化（タップで詳細表示）

**設計上の論点（次セッションで深い議論を予定）:**
- Map 16方位スコアと地点別ハウスの統合方法
  - A: 16方位スコアを廃止し、地点ごとの完全リチャートに切替
  - B: 16方位スコアは維持し、地点別ハウスを「補助情報」として表示
  - C: ズームレベルに応じて切替（広域=16方位 / 拡大=地点別リチャート）
- パフォーマンス（地点ごとに houses 計算が走る）
- アスペクト線（MC line / IC line / ASC line / DSC line）の地図上描画
- アストロカートグラフィー専門用語の UI 整理
