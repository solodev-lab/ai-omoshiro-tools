
ターミナルの場所について：オーナーが普段使っている**PowerShellやcmd**から
`
```
cd E:\AppCreate\apps\solara
```

` して ````
```
flutter emulators --launch Pixel_8
```
を実行すれば、そのターミナルで `r` / `R` キーが使えます。今回は僕のバックグラウンドプロセスで起動したので、オーナー側のターミナルからは操作できません。

**リロードの方法：**

```
flutter run
```
- 
- ` を実行したPowerShellウィンドウで `r` キーを押す → ホットリロード
- `R` キー → ホットリスタート（状態リセット）


メモリリセット
何度も修正しているとメモリ不足のエラーがでる。でたら。
PowerShellで以下を実行してください：

```
C:\Users\cojif\AppData\Local\Android\Sdk\platform-tools\adb.exe emu kill
```

エミュレータが閉じたら：

```
C:\Users\cojif\AppData\Local\Android\Sdk\emulator\emulator.exe -avd Pixel_8 -wipe-data
```

その後：

```
flutter run
```


これはFlutterのデバッグ接続がAndroidエミュレータ/デバイスと繋がらない問題。よく起こる。

**以下を順番に試してください:**

1. まず `Ctrl+C` でFlutterを停止
    
2. ADBを再起動:
    



```
cd C:\Users\cojif\AppData\Local\Android\Sdk\platform-tools\
```

```
.\adb.exe kill-server
.\adb.exe start-server
```

```
cd E:\AppCreate\apps\solara
```

3. 再度実行:

```
flutter run
```

それでもダメな場合は、エミュレータを一度閉じて再起動してから `flutter run` してください。
