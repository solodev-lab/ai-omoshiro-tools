# Solara リリースチェックリスト

> ストアに出す前に、このリストを上から順に全部確認する。
> 1つでも「NO」があればリリースしない。

---

## 1. 動作確認

- [ ] 全5画面が正常に表示される（Map/Horo/Tarot/Galaxy/Sanctuary）
- [ ] 画面間のナビゲーション（ボトムナビ）が正常に動く
- [ ] プロフィール未設定の状態でもクラッシュしない
- [ ] プロフィール設定→保存→再起動で復元される
- [ ] 地名検索（Nominatim）が動作する
- [ ] オフライン状態でアプリがクラッシュしない（API失敗時のエラー処理）
- [ ] 端末の戻るボタンでアプリが意図通りに動く（Android）

## 2. パフォーマンス

- [ ] 起動時間が3秒以内
- [ ] 画面遷移がスムーズ（カクつきなし）
- [ ] メモリリークがない（長時間使用でもクラッシュしない）
- [ ] 地図画面のスクロール/ズームがスムーズ

## 3. 端末対応

- [ ] 小画面（5インチ）で表示が崩れない
- [ ] 大画面（タブレット）で表示が崩れない
- [ ] ダークモード/ライトモード切替で問題ない
- [ ] 横向き回転でクラッシュしない（または回転ロック済み）
- [ ] ノッチ/パンチホールでUIが隠れない

## 4. ストアポリシー（Google Play）

- [ ] プライバシーポリシーのURLが設定されている
- [ ] アプリ内にプライバシーポリシーへのリンクがある（Sanctuary画面）
- [ ] 占い結果に免責文言がある（「娯楽目的であり、医療・金融等の助言ではありません」）
- [ ] パーソナルデータの使用目的を説明している
- [ ] ターゲット年齢層の設定が正しい（占いアプリは13歳以上推奨）
- [ ] アプリの説明文に誤解を招く表現がない（「100%当たる」等はNG）
- [ ] スクリーンショットが実際のアプリと一致している

## 5. セキュリティ

- [ ] APIキーがソースコードにハードコードされていない
- [ ] デバッグログが本番ビルドで無効化されている
- [ ] ProGuard/R8 が有効（Android リリースビルド）
- [ ] HTTPSのみ使用（HTTP通信なし）

## 6. 分析・監視（リリース前に導入必須）

- [ ] Firebase Analytics が導入済み
- [ ] Firebase Crashlytics が導入済み
- [ ] 初回起動イベントが送信されることを確認
- [ ] クラッシュが Crashlytics ダッシュボードに表示されることを確認

## 7. ビルド・署名

- [ ] リリース用の署名キー（keystore）が作成済み
- [ ] キーストアのパスワードを安全な場所に保管した
- [ ] バージョン番号を更新した（pubspec.yaml の version）
- [ ] リリースビルドが成功する（`flutter build appbundle`）
- [ ] リリースAPKの実機テストが完了

## 8. ストア掲載情報

- [ ] アプリ名（日本語/英語）
- [ ] 短い説明文（80文字以内）
- [ ] 詳細な説明文（4000文字以内）
- [ ] スクリーンショット 最低4枚
- [ ] フィーチャーグラフィック（1024x500）
- [ ] アプリアイコン（512x512）
- [ ] カテゴリ選択（ライフスタイル or エンタテインメント）
- [ ] コンテンツレーティング質問票の回答
