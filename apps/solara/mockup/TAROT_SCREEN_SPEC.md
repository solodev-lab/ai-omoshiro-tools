# Solara タロット画面 仕様書

> Action画面（地図）とは別に設けるタロット専用画面の仕様。

## 1. 画面構成

### ナビゲーション
- 下部ナビの「Rhythm」または専用タブからアクセス
- タロット画面内で以下を切替:
  - **ネイタルタロット**（タブ1）
  - **トランジットタロット**（タブ2）

## 2. ネイタルタロット

### 概要
- **1日1回**引ける
- 「今日のあなた」のエネルギー・方位を占う
- 現在のSeed Card機能がこれに該当

### カードドロー
- カード裏面表示 → タップで3Dフリップ → 結果表示
- 結果はAction画面（地図）にも反映される
  - 方位セクターの活性化（blessed/shadow/mid）
  - 惑星ラインのブースト表示

### 履歴
- タロット画面内に履歴セクションを設置
- 表示内容:
  - 日付
  - 引いたカード名・シンボル
  - その日の方位結果（吉方位・要注意方位）
- 単純な過去ログとして閲覧

## 3. トランジットタロット

### 概要
- **任意のタイミング**で引ける
- **日付 × 場所**を指定して「未来の予定」を占う
- トランジット（天体の実際の通過位置）に基づくブースト方位を算出

### 入力
1. **日付選択**: カレンダーUIまたは日付ピッカー
2. **場所指定**: 以下のいずれか
   - 地図タップ（座標取得）
   - 検索（Nominatimジオコーディング）
   - GPS現在地

### カードドロー
- 入力後にカードを引く
- 結果: 指定日・指定場所でのトランジット天体配置に基づくブースト方位

### 結果表示
- カード名・シンボル
- 指定場所の地図上にブースト方位を表示
- 「この日・この場所ではこの方角にエネルギーが集中」

### 履歴 / 予定表機能
- タロット画面内に履歴セクションを設置
- **過去の記録**と**未来の予定**の両方を含む
- 表示内容:
  - 日付（過去 or 未来）
  - 場所名・座標
  - 引いたカード名・シンボル
  - ブースト方位
- **未来の予定はいつでも見返せる**
  - 旅行計画の参考として機能
  - 「3/28 京都 → 北東にブースト」のように確認できる
- 時系列で並べることで予定表のように使える

## 4. 技術要件

### 座標取得
```javascript
// GPS（現在地）
navigator.geolocation.getCurrentPosition(pos => {
  const lat = pos.coords.latitude;
  const lng = pos.coords.longitude;
});

// 地図タップ
map.on('click', e => {
  const lat = e.latlng.lat;
  const lng = e.latlng.lng;
});

// 検索（Nominatim）
fetch('https://nominatim.openstreetmap.org/search?format=json&q=' + query)
```

### トランジット天体位置（将来実装）
- 天文計算ライブラリ（astronomy-engine等）でリアルタイム天体位置取得
- 指定日時のトランジット天体配置を計算
- ユーザー座標 × 天体位置 → ブースト方角算出

### データ保存
- 履歴データはローカルストレージまたはバックエンド（将来）
- 保存項目:
  ```json
  {
    "type": "natal" | "transit",
    "date": "2026-03-26",
    "card": { "name": "CUP III", "symbol": "🌊", "subtitle": "再会の喜び" },
    "location": { "name": "京都", "lat": 35.0116, "lng": 135.7681 },
    "boostDirection": { "label": "NE", "angle": 45 },
    "sectorResults": { ... }
  }
  ```

## 5. Action画面との連携
- ネイタルタロットの結果 → Action画面の地図に自動反映
- トランジットタロットの結果 → Action画面で「プレビュー表示」可能
  - 指定場所を中心にした地図表示 + ブースト方位のオーバーレイ

## 6. 優先度
- **Phase 1**: ネイタルタロット（現行Seed Card機能の移設 + 履歴）
- **Phase 2**: トランジットタロット（日付×場所指定 + 履歴/予定表）
- **Phase 3**: トランジット天体計算の実装（astronomy-engine統合）
